Introduction

"xev" มาจาก next generation dev (ก่อนหน้านี้ตั้งชื่อว่า xdev แต่เพื่อให้สั้นอีก จึงเป็น xev)... เป็นแนวคิดที่จะทำให้การ dev ง่ายขึ้น คนทั่วไปเข้าใจได้ไม่ยาก ทำให้เกิด clean & clear code

Pattern ของคำสั่ง หรือจะเรียกว่า ภาษา ก็พอได้ เป็นดังนี้

xev({...})
หรือ xev({do:this, to:whom, by:how})
ทำนองนี้ จะเห็นว่า รูปแบบจะเป็น object แต่จะเป็น object ที่เรียงกันคล้ายภาษาคน

ยกตัวอย่างจะส่งข้อมูลให้ใครคนหนึ่ง ก็อาจใช้ว่า xev({send:customerData, to:"john"}) เป็นต้น ดังนั้นอาจจะพูดได้ว่า xev จะทำให้เกิดภาษาทางซอฟท์แวร์ที่ คล้ายคน มากขึ้น ก็จะพยายามพัฒนาจากจุดนี้เป็นต้นไป ไม่แน่ว่าอาจจะไปถึงขั้น AI ก็เป็นได้

เบื้องต้นจะทำใน java script ก่อน โดยหลักแล้วเราไม่ได้คิด engine ของภาษาขึ้นมาใหม่ เพียงแต่เอาภาษาเดิมนั่นแหละ เช่น java script มาทำให้ดูง่ายขึ้น เท่านั้น แล้วหากทำสำเร็จที่ js ก็จะขยายไปภาษาอื่นๆ ต่อไป โลก อาจกลายเป็นว่า ไม่ว่าจะภาษาอะไร เช่น python, java, C, R หรืออะไรก็แล้วแต่ ก็สามารถ "พูดคุย" กันได้ เพราะคุยกันผ่าน xev นั่นเอง

Specification

  1. เป้าหมายคือจะให้ xev เป็น ภาษา หรือ โมเดล หรือรูปแบบ ที่ใช้เหมือนกันในทุกๆ programming language และทำให้เกิด code ของซอฟท์แวร์ในลักษณะ clean & clear ไม่สลับซับซ้อน จนคนทั่วไปเข้าใจได้ยาก
  2. ใช้ภาษาอังกฤษ (English) ในการเขียน xev
  3. รูปแบบคือ xev({...}) โดยจะส่ง input ให้ xev function ที่เป็น object
